# P5-T14 — Reconcile the Specification Lifecycle

## Objective

Make PRD status, hardening semantics, open-question decisions, and successor release gates authoritative and traceable.

## Acceptance Criteria

- PRD status/version reflect post-release hardening.
- Canonical semantics for incomplete/unknown, zero scope, policy composition, root/sudo, selectors, and release metadata are explicit.
- A release acceptance checklist links implementation gates to the successor release.

## Dependency

P5-T1 — Audit v0.1.0 Release.
