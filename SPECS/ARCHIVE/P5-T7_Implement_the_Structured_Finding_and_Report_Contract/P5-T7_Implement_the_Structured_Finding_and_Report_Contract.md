# P5-T7 — Implement the Structured Finding and Report Contract

## Objective

Make every finding machine-readable and human-actionable with stable identity, severity, state comparison, risk, and scope.

## Acceptance Criteria

- Report records include rule ID, severity, current/expected state, risk, and scope.
- Rendered diagnostics sanitize control characters and preserve stable status/exit behavior.
- Metadata, ancestor, and ACL findings map to explicit rules.

## Dependency

P5-T3, P5-T4, P5-T5, P5-T6.
